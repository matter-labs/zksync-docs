ZKsync is an open-source project. We champion community-driven development, which means you,
from any corner of the world, can contribute to shaping ZKsync's future.

## Open-source repositories

Here's a list of our key open-source repositories that you can contribute to:

### ZKsync OS

- [**ZKsync OS**](https://github.com/matter-labs/zksync-os): Generalized RISC-V based state transition function for the ZKsync protocol.
- [**ZKsync OS Server**](https://github.com/matter-labs/zksync-os-server): Server implementation for ZKsync OS.
- [**ZKsync OS Scripts**](https://github.com/matter-labs/zksync-os-scripts): Scripts for ZKsync OS protocol upgrades.

### Smart Contracts

- [**era-contracts**](https://github.com/matter-labs/era-contracts): Submodule containing the smart contracts for ZKsync.

### Prover & Protocol

- [**Airbender**](https://github.com/matter-labs/zksync-airbender): RISC-V prover system for ZKsync OS.
- [**zksync-protocol**](https://github.com/matter-labs/zksync-protocol): Houses the ZKsync Era protocol implementation, including VM and circuits.

### Development tools

- [**Airbender Platform**](https://github.com/matter-labs/airbender-platform): Rust toolkit for writing RISC-V zk-provable programs.
- [**anvil-zksync**](https://github.com/matter-labs/anvil-zksync): An in-memory node designed for integration testing and debugging.

### Explorer

- [**block-explorer**](https://github.com/matter-labs/block-explorer): The official block explorer for navigating the ZKsync network.

Feel free to explore these repositories, and don't hesitate to contribute!
