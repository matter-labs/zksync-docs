# Reviewer Rubric: UX and Learnability

## Blocking Findings

- prerequisites missing for required steps
- critical step order causes likely user failure
- verification of success is absent for core flows

## Non-Blocking Suggestions

- improve step clarity and outcome statements
- add troubleshooting for common failure points
- reduce cognitive load with better section breaks

## Review Output Format

- `Blocking issues`
- `Non-blocking suggestions`
- `Suggested patch snippets` (small, optional)
- `Confidence: high|med|low`
