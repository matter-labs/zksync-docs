# Reviewer Rubric: Correctness

## Blocking Findings

- factual claims lack supporting sources
- docs contradict primary repo sources
- undocumented assumptions presented as facts
- invented APIs, flags, versions, or behavior

## Non-Blocking Suggestions

- add stronger citations for important claims
- tighten ambiguous wording
- improve verification guidance

## Review Output Format

- `Blocking issues`
- `Non-blocking suggestions`
- `Suggested patch snippets` (small, optional)
- `Confidence: high|med|low`
